# Devlog

This directory is the working documentation area for ongoing development.

## Structure

- `planning/`: current planning, active problem analysis, tradeoff records, and backlog
- `appendix/`: stable reference material for the current system
- `archive/`: frozen snapshots of finished or baseline versions

## Usage

- Put active version planning in `planning/` without embedding the version in filenames.
- When a version is closed, snapshot the relevant planning and appendix materials into `archive/vX.Y.Z/`.
- Keep `appendix/` focused on durable reference material rather than daily progress notes.

## Current baseline

- Archived baseline version: `v0.1.0`
- Current planning target: `v0.2`
