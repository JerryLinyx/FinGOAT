# v0.2.0 Chart Query History Requirements

## Status Note

本文件是早期需求稿，不再代表当前主线现状。

当前状态请同时参照：

- `ADR-022`
- `ADR-033`

## 1. 背景

当前 Chart 页面可按股票代码查询并查看行情，但缺少“历史查询入口”，导致重复输入成本高，常查标的切换效率低。

## 2. 新增需求（本次追加）

在 Chart 页面增加“历史夹/历史列表”能力：

1. 记录用户查过的股票代码。
2. 历史记录去重（同一代码仅保留一条）。
3. 若再次查询已存在代码，则该代码置顶（按最近查询时间排序）。

## 3. 已确认事实

- 现有 chart 相关能力已覆盖行情查询与参数控制。
- 前端当前未提供独立的“历史股票代码列表”交互。
- `推断`：若以用户维度持久化，需新增后端接口或在现有用户配置模型中扩展字段。

## 4. 方案对比（摘要）

### 方案 A：仅前端本地存储（LocalStorage）

- 优点：实现快、无后端改动
- 缺点：跨设备不同步，清缓存后丢失

### 方案 B：后端持久化 + 前端缓存（推荐）

- 优点：可跨端同步、可用于后续个性化推荐
- 缺点：需要接口与数据模型改动

## 5. 决策

先按“前端可用、接口可扩展”设计推进：  
MVP 可先本地落地，接口预留后端持久化扩展位。

## 6. 当前状态

- 状态：历史需求稿；本地历史查询能力已实现，剩余缺口是跨端同步
- 关联 backlog：`current/task-backlog.md` 中保留对应后续条目
